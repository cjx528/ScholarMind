# ScholarMind 文档索引

这里保留当前版本真正需要维护的文档。已经不对应代码的旧渠道实验、旧阅读器方案和历史计划文档已删除，避免开发和答辩时引用错误入口。

## 推荐阅读顺序

1. [CURRENT_STATE.md](CURRENT_STATE.md)：先了解当前版本有哪些功能、哪些旧功能已经移除。
2. [FUNCTION_INVENTORY.md](FUNCTION_INVENTORY.md)：逐项列出所有具体功能、入口、接口和验收点。
3. [TEAM_VALIDATION.md](TEAM_VALIDATION.md)：小组成员按 A/B/C/D 认领功能测试和修复范围。
4. [TECH_DETAILS.md](TECH_DETAILS.md)：查看接口、数据流、服务模块和主要文件位置。
5. [COURSE_PRESENTATION_GUIDE.md](COURSE_PRESENTATION_GUIDE.md)：课堂演示路线。
6. [PRESENTATION_DRAFT_CURRENT.md](PRESENTATION_DRAFT_CURRENT.md)：报告和答辩框架。
7. [REFERENCE_PIPELINE_NOTES.md](REFERENCE_PIPELINE_NOTES.md)：推荐发现流水线和阅读助手的后续改进思路。
8. [deployment/DOCKER_DEPLOYMENT.md](deployment/DOCKER_DEPLOYMENT.md)：需要部署时再看。

## 当前主线

- 推荐发现：画像表单、AI 方向细分、其他方向自由填写、推荐队列、反馈学习、研究雷达分区。
- 论文收集：单源入库、多源搜索、渠道建议、主题订阅、关键词建议、手动抓取、CS 分类订阅。
- 阅读理解：论文库筛选、标签、收藏、批量处理、论文详情、PDF 下载和阅读、粗读、精读、推理链、相似论文、解析追问。

## 文档维护原则

- 文档必须以当前代码为准。
- 如果某个功能入口已经从侧边栏或路由中删除，不要继续写入新文档。
- 如果修改了 README 或 docs，也要检查是否还残留旧项目名、旧接口名和旧测试文件名。
