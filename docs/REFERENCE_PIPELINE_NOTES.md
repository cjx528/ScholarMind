# 外部推荐发现流水线参考

这份文档记录已经吸收到 ScholarMind 当前路线中的参考点。它不是实现说明，也不是要求照搬外部代码。

## 最值得迁移的方向

最有价值的是“推荐发现流水线”，不是 UI 或部署架构。

当前版本已经把主线调整为：

- 每天自动生成研究雷达。
- 将候选论文分成精读候选、速读候选和跳过原因。
- 将主题订阅升级成更接近 query profile 的结构。
- 让论文解析结果可以继续追问。

## 推荐发现流水线

理想流水线：

1. BM25 召回：用关键词快速召回相关论文。
2. Embedding 召回：用语义相似度补充关键词漏掉的论文。
3. RRF 融合：把多路召回结果融合，降低单一路径偏差。
4. Reranker 重排：用更强模型重新评估相关性。
5. LLM refine：把排序结果转成可读理由和分区。
6. 最终分区：精读候选、速读候选、跳过原因。

当前代码已有研究雷达和多源收集基础，后续可以逐步补强召回融合和重排解释。

## 主题订阅升级方向

一个主题不应该只是关键词列表。更合理的 query profile 应包含：

- 主题名称。
- 关键词。
- 意图 query。
- 论文源选择。
- 启停状态。
- 最近运行结果。
- 推荐反馈统计。

当前相关代码：

- `frontend/src/pages/Collect.tsx`
- `frontend/src/contexts/ChannelContext.tsx`
- `apps/api/routers/topics.py`
- `packages/ai/daily_runner.py`
- `packages/integrations/`

## 阅读助手升级方向

阅读助手不应只生成一次性报告，而应该支持持续追问。

当前已经落地：

- PDF 选中文本后可翻译成中文、解释、总结和提问。
- 论文详情的粗读、精读、推理链可进入同一套追问上下文。
- 单篇论文统一走 `POST /papers/{paper_id}/ask`。

后续可以继续补：

- 当前页面会话持久化。
- 最近问题。
- 置顶问题。
- 解析追问历史。

## 不迁移的部分

- 不照搬外部 UI。
- 不照搬部署结构。
- 不恢复旧双栏译文页。
- 不恢复旧视觉说明入口。

ScholarMind 的重点是“推荐发现 + 阅读追问 + 知识沉淀”的闭环。
