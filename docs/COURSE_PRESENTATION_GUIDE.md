# 课堂展示路线

目标：让老师看到 ScholarMind 不是一个静态论文列表，而是一个从发现、阅读、追问到沉淀的完整科研助手。

## 展示前准备

1. 确认 `.env` 已配置可用 LLM Key。
2. 启动后端：

```powershell
.\.venv\Scripts\python.exe -m uvicorn apps.api.main:app --host 127.0.0.1 --port 8000 --reload
```

3. 启动前端：

```powershell
cd frontend
npm.cmd run dev -- --host 127.0.0.1
```

4. 打开：

- 前端：`http://127.0.0.1:5173`
- 后端健康检查：`http://127.0.0.1:8000/health`

## 推荐展示顺序

### 1. 用户画像

页面：`/recommendation`

展示重点：

- 研究方向可以细分到 LLM、MLLM、Agent、图像生成/编辑、视频理解/生成、语音交互、AI4Science、高效推理/评测等。
- 也允许填写其他非 AI 方向。
- 全局 LLM 设置在设置页管理，不把模型配置塞进画像表单。

### 2. 论文收集

页面：`/collect`

展示重点：

- 支持多源渠道选择，搜索时可直接选择 arXiv、OpenReview、Semantic Scholar、OpenAlex、DBLP、bioRxiv。
- 搜索结果先作为候选展示，需要人工勾选确认后才入库。
- 候选论文会预判主题，入库后自动归入相应主题库。
- OpenReview 和 arXiv 可以作为现场演示重点，因为容易解释“最新论文发现”的价值。

### 3. 论文库

页面：`/papers`

展示重点：

- 搜索、筛选、排序、收藏和标签。
- 进入论文详情页后，展示 PDF 下载、粗读、精读、基于画像的个性化分析、推理链和相似论文。

### 4. 论文详情与解析追问

页面：`/papers/:id`

展示重点：

- 粗读用于快速判断论文是否值得读。
- 精读用于方法、实验、风险和创新点分析，并包含基于用户画像的个性化分析。
- 推理链用于看清论文论证结构。
- 解析结果不是终点，用户可以继续追问，例如“审稿风险的证据是什么”。

### 5. PDF 阅读助手

入口：论文详情页的 PDF 阅读器。

展示重点：

- 选中英文段落后点击“翻译成中文”。
- 继续问“这段为什么重要”或“作者这里想解决什么问题”。
- 没有选中文本时，也能问整篇论文或已有解析。

- 读过的论文可以沉淀成 Wiki。

### 8. Agent、设置和运维

页面：`/settings`、`/dashboard`、`/statistics`

展示重点：

- Dashboard 和 Statistics 展示系统任务、论文状态和成本统计。
- Agent 可以作为补充展示，不要占用主线过多时间。

## 不建议展示的内容

- 不展示已经移除的旧双栏译文页。
- 不展示已经隐藏的旧视觉说明入口。
- 不展示不在当前侧边栏中的历史实验页面。

## 备用讲法

一句话介绍：

> ScholarMind 把论文推荐、论文阅读和阅读后的知识沉淀串起来，目标是让用户每天知道该读什么，读的时候能直接问，读完之后能留到自己的研究知识库里。

三段式介绍：

1. 论文发现：用画像偏好辅助多源搜索、候选审核和主题归档。
2. 阅读理解：在论文详情和 PDF 阅读器里生成粗读、精读、推理链，并支持继续追问。
